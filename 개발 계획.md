# 교육심리 논문 위젯 앱 — 개발 계획 (v3: 백엔드 0, LLM 0)

> Journal of Educational Psychology의 인용수 상위 논문 초록을 매일 1편씩 보여주는 안드로이드 앱.
> **백엔드 없음. LLM API 없음. 운영비 $0, 빌드 비용 $0.**
> 한국어는 ML Kit 온디바이스 번역으로 런타임 처리.
> 홈 화면 위젯 + 메인 앱.

---

## 0. 적용 철학 — Karpathy Style

| # | 원칙 | 이 프로젝트에서 |
|---|---|---|
| 1 | **Become one with the data** | 코드/UI 전에 OpenAlex 응답을 직접 본다 (Phase 0 — 일부 완료) |
| 2 | **End-to-end dumb baseline** | "JSON 1개 → 화면 1개에 오늘 카드 1장" 먼저 관통 |

| 5 | **Eval before iteration** | "좋은 요약" 점수가 있어야 프롬프트 튜닝 |

## 1. 프로덕트 정의

- **누구**: 교육심리에 관심 있는 한국어 사용자
- **무엇**: 매일 1편 — 영어 원제, 한국어 번역 제목, 한국어 3~5문장 요약, 핵심 용어 5개, 발행연도, 인용수, DOI 링크
- **표면**:
  - **메인 앱**: 한 화면에 오늘 카드 한 장 + 어제로 가기/내일로 가기 + 히스토리
  - **홈 화면 위젯**: 오늘 카드의 제목+요약 1~2문장 (탭 시 메인 앱)
- **왜**: 영어 논문 진입장벽을 낮춘다. "오늘 한 편"이라는 강제된 적은 양으로 부담을 줄인다.
- **성공 기준 (V1)**:
  - 30일 retention ≥ 25%
  - 위젯 설치율 ≥ 30% (메인 앱 설치자 중)
  - 자체 eval set 30건 요약 품질 ≥ 4.0/5

비목표 (Out of scope, V1):
- 본문 PDF 제공 (저작권)
- 다른 저널 추가
- 사용자 계정·로그인
- 클라우드 동기화
- iOS

## 2. 아키텍처 (백엔드 0, LLM 0)

```
[ 개발자 PC (빌드 타임, 1회) ]
   ├─ scripts/fetch_openalex.py     →  OpenAlex API (500편)
   ├─ scripts/score_and_filter.py   →  age-normalized 정렬, 2021 이전, DOI dedup → top 365
   └─ scripts/build_assets.py       →  슬림 JSON 생성
                ↓
   assets/papers.json (365편, 영어 원문, 553 KB)
                ↓
       [ APK assets 번들 ]
                ↓
[ 사용자 폰 ]
   ├─ 메인 앱 (Compose)
   │    ├─ 첫 실행 시 papers.json → Room DB 시드
   │    ├─ 오늘 카드 (영어 제목 + 영어 초록)
   │    └─ [한국어로 보기] 버튼 → ML Kit 번역 (온디바이스)
   ├─ 홈 화면 위젯 (Glance)
   │    └─ 영어 제목 + 인용수 + 연도 (위젯엔 번역 안 함, 짧으니까)
   └─ WorkManager
        └─ 자정마다 위젯 갱신
```

**런타임 외부 호출 = 0건.** API 키 0개. ML Kit 번역 모델은 첫 사용 시 한 번만 다운로드(~30MB), 이후 완전 오프라인.

## 3. 데이터 파이프라인 (빌드 타임, LLM 없음)

### 3.1 데이터 소스
- **OpenAlex** (무료, 키 불필요)
- 필터: `primary_location.source.issn:0022-0663|1939-2176`

### 3.2 정렬 — age-normalized
- 점수 = `cited_by_count / max(2026 - year, 3)`
- **2021년 이전 발행만 후보** (5년 이상 누적 시간 보장)

### 3.3 필터링 (코드 1줄짜리 룰만)
- 초록 결측 제외
- DOI 기준 중복 제거
- `type == "article"`만

(*v2의 LLM 관련성 분류·한국어 요약·eval 단계는 D 결정으로 모두 제거됨.*)

### 3.4 산출물 — 이미 완료
- `data/raw_top500.json` — OpenAlex 500편 원본
- `data/candidates_365.json` — 필터·정렬 후 365편
- **`assets/papers.json`** (553 KB) — APK 번들용 슬림 버전

스키마:
```json
{
  "version": "2026-05-08",

      "citedBy": 7724,
      "scorePerYear": 214.6,
      "authors": ["Pintrich", "De Groot"],
      "oaPdfUrl": null
    }
  ]
}
```

### Phase 2 — 메인 앱 MVP (3~5일) ← 다음 단계

스택: Kotlin + Jetpack Compose + Room + WorkManager + **ML Kit Translation**

화면:
- **오늘 카드**: 영어 제목, 영어 초록, 인용수, 연도, 저자, DOI 링크, **[한국어로 보기]** 토글 버튼
- **이전/다음 카드**: 좌우 스와이프로 day-1, day+1
- **히스토리**: 최근 30일 리스트
- **설정**: 없음 (V1)

선정 로직:
```kotlin
val installEpochDay = prefs.getLong("installEpochDay", -1)  // first launch에 설정
val today = LocalDate.now(ZoneId.systemDefault()).toEpochDay()
val dayIndex = ((today - installEpochDay) % papers.size).toInt()
val todayPaper = papers[dayIndex]  // already sorted by scorePerYear desc
```

ML Kit 번역:
```kotlin
val translator = Translation.getClient(
    TranslatorOptions.Builder()
        .setSourceLanguage(TranslateLanguage.ENGLISH)
        .setTargetLanguage(TranslateLanguage.KOREAN)
        .build()
)
translator.downloadModelIfNeeded().await()  // ~30MB, 첫 1회
val ko = translator.translate(abstract).await()
```

|---|---|---|
| **데이터 갱신 = APK 재배포** | 풀 갱신 느림 | 분기별 또는 반기별. v2부터 정적 호스팅(GitHub Pages) 검토 |

| Hotelling/Rubin 거장 논문 섞임 | 사용자 기대 미스매치 | UI에 발행연도·인용수 명시. 사용자가 스스로 판단. (D 결정으로 자동 분류 X) |
| 인용수 편중 (오래된 논문) | 다양성 ↓ | age-normalized 정렬 + 2021 이전 필터 |
| 초록 결측 13.6% | 풀 크기 ↓ | 500편→365편 확보로 해소 |
| **저작권 (영어 초록 표시)** | **법적 리스크** | 학술 인용 범위 추정. DOI 링크 노출. 출처 명시. **법무 검토 권장** |
| 폰 시간/타임존 변경 시 인덱스 점프 | 사용자 혼란 | 설치일 epochDay 저장, UTC 기준 day delta |
| ML Kit 모델 30MB 다운로드 | 첫 사용 시 지연 | 첫 실행 시 백그라운드 미리 다운로드 옵션 |
