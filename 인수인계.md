## 3. 남은 작업 (조카가 진행)

### A. Android APK release 배포 — 🔵 진행 중 (build.gradle 까지 셋업 완료)

이미 처리됨:
- `app/build.gradle.kts` 에 `signingConfigs.release` 추가, `local.properties` 4 항목 읽도록 구성
- `.gitignore` 에 `*.jks` 추가

조카가 할 일:
1. **Keystore 생성** (1회):
   ```powershell
   keytool -genkey -v -keystore release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias cherin-release
   ```
   `keytool` 인식 안되면 JDK bin 절대경로:
   ```powershell
   & "C:\Program Files\Microsoft\jdk-17.0.18.8-hotspot\bin\keytool.exe" -genkey -v -keystore release.jks ...
   ```
   비밀번호 입력. 나머지 항목은 영문 (이름·도시·국가코드 KR).

2. **`local.properties` 끝에 4줄 추가**:
   ```properties
   RELEASE_STORE_FILE=release.jks
   RELEASE_STORE_PASSWORD=비밀번호
   RELEASE_KEY_ALIAS=cherin-release
   RELEASE_KEY_PASSWORD=비밀번호
   ```
   `local.properties` 는 gitignored — push 안 됨.

3. **Release APK 빌드**:
   ```powershell

   gradle assembleRelease
   ```
   (조카 PC 에서는 위 gradle 경로가 다를 수 있음 — 4 절 참고.)
   결과: `app/build/outputs/apk/release/app-release.apk` (~30 MB)

4. **GitHub Releases 에 업로드**:
   - https://github.com/108nevermind-web/cherin/releases/new

   - Publish release
   - 다운로드 직접 링크: `https://github.com/108nevermind-web/cherin/releases/download/v0.1.0/app-release.apk`

## 5. 다른 계정·PC 작업 시 고려사항

### 5-1. Keystore (가장 중요)
- **첫 release APK 를 본인 PC 에서 만들어 보지 않은 상태**라면, 조카 PC 에서 새로 만들어도 됨 (이게 첫 release)
- **본인 PC 에서 이미 release.jks 만들었으면**, 그 파일을 USB 또는 암호화 메일로 조카에게 안전하게 전달. 조카 PC 의 프로젝트 루트에 같은 이름으로 저장
- **둘 다 새 keystore 만들면 안 됨**: 다른 keystore = 다른 앱 identity = 사용자가 새 APK 를 기존 앱 위에 못 깔림 (반드시 기존 앱 삭제 후 설치)

### 5-2. GitHub Pages 도메인
- collaborator 방식 (방법 1): URL 그대로 → 가족 카톡 그대로 유효
- fork 방식 (방법 2): URL 바뀜 → 가족·친구 다시 안내 필요. 비추천.

### 5-3. Git 사용자 정보
조카 PC 에서 첫 commit 전:
```powershell
git config --global user.name "조카 이름"
git config --global user.email "조카 이메일"
```

### 5-4. 작업 내용 동기화
- 본인 (108nevermind-web) 이 코드 수정 push → 조카는 `git pull` 로 받음
- 충돌 방지: 둘이 동시에 같은 파일 수정 안 하기
- 큰 작업은 branch 분리: `git checkout -b feature/xxx` → push → GitHub 에서 PR 생성 → main 에 merge

## 8. 비밀 정보 / 백업 권장

| 항목 | 위치 | 백업 |
|---|---|---|
| `release.jks` | 프로젝트 루트 (gitignored) | USB + 비공개 클라우드 (구글드라이브 등) — 분실 시 영원히 업데이트 못 함 |

| GitHub Personal Access Token (push 시 필요) | https://github.com/settings/tokens | 패스워드 매니저 |

### 분실 시 복구
- **keystore 분실**: 새 keystore 로 새 applicationId 의 새 앱 빌드 → 사용자가 기존 앱 지우고 다시 설치 → 즐겨찾기·설정 다 사라짐. **회피 불가능**.
- **GitHub 토큰 분실**: 새로 발급, 토큰은 만료시 재발급 필요한 항목.

## 9. 자주 막히는 곳

| 증상 | 원인 | 해결 |
|---|---|---|
| `gradle.bat` 인식 안됨 | PATH 미설정 | `$env:PATH += ";...gradle-9.4.1\bin"` 또는 절대경로 호출 |
| Android Studio 가 JDK 21 다운로드 시도 | `gradle/gradle-daemon-jvm.properties` 의 `toolchainVersion` 확인 | 17 로 고정되어 있어야 함 (이미 셋업됨) |
| `keytool` not found | JDK PATH 빠짐 | `System.getProperty("java.home")` 또는 절대경로 |
| Release APK 빌드 시 "no signing config" | local.properties 의 4 줄 누락 | 3절 A-2 다시 |
| GitHub push 시 "Authentication failed" | 토큰 만료 / 다른 계정 | https://github.com/settings/tokens 새 토큰, repo 권한 체크 |
| GitHub Pages 가 옛 데이터 표시 | 서비스 워커 캐시 | PWA 의 ⚙️ → "데이터 새로고침" 버튼, 또는 사용자가 브라우저 강력 새로고침 (Ctrl+Shift+R) |
| ML Kit 번역 모델 다운로드 실패 | Google Play Services 미작동 / 네트워크 | 에뮬레이터는 Google Play 포함 image 사용. 번역 실패 시 영어 원문은 정상 표시되도록 graceful 처리됨 |
