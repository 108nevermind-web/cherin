## 3. 로컬 git 초기화 + 첫 push

PowerShell 또는 Git Bash 에서 프로젝트 루트로 이동:

```powershell
cd D:\AI_Root\cherin
```

처음 한 번만 git 사용자 정보 (이미 설정되어 있으면 스킵):
```powershell
git config --global user.name "Your Name"
git config --global user.email "108nevermind@gmail.com"
```

repo 초기화 + 첫 commit + push:

```powershell
git init -b main
git add .
git status      # 어떤 파일들이 들어가는지 확인 (local.properties, build/, notes/ 는 제외되어 있어야 함)
git commit -m "initial: Android app + PWA mirror"
git remote add origin https://github.com/108nevermind-web/cherin.git
git push -u origin main
```

push 시 GitHub 에서 **Personal Access Token** 을 비밀번호로 요구할 수 있어요:
- https://github.com/settings/tokens → **Generate new token (classic)** → repo 권한 체크 → 생성
- 토큰 문자열을 비밀번호 자리에 붙여넣기

## 4. GitHub Pages 활성화

1. push 한 repo 페이지 (예: `https://github.com/108nevermind-web/cherin`) 로 이동
2. 상단 탭 **Settings** → 좌측 메뉴 **Pages**
3. "Source" → **Deploy from a branch**
4. "Branch" → **main** 선택, 폴더는 **/docs** 선택
5. **Save** 클릭
6. 1~2분 후 같은 화면 상단에 발행 URL 표시:
   ```
   ✅ Your site is live at https://108nevermind-web.github.io/cherin/
   ```

## 5. PC 브라우저에서 1차 검증

1. 위 URL 을 Chrome 으로 열기
2. 정상 로드되면 "Day N / 365" + 논문 카드가 보여야 함
3. **F12 → Console** 탭: 빨간 오류 없는지 확인
4. **F12 → Application → Service Workers**: `sw.js` 가 `activated` 상태인지 확인
5. **F12 → Application → Manifest**: 아이콘과 이름 표시 확인
6. 각 기능 시험:
   - 다음/이전 버튼 → 다른 논문 표시
   - ☆ 별 버튼 → ★ 채워짐, 페이지 새로고침 후에도 유지 (localStorage)
   - "목록" → 검색·즐겨찾기 탭 작동
   - "번역 (Papago)" → 새 창에 Papago 가 영어 → 한국어 번역 결과 노출
   - "원문 (DOI)" → doi.org 새 창

문제 발견 시 캡처해서 알려주세요. 제가 수정해서 commit + push 하면 됩니다.

## 6. iPhone 에서 직접 시험

1. iPhone Safari 에서 같은 URL 열기
2. 화면 하단 가운데 **공유** 버튼 (↑ 아이콘) 누르기
3. 메뉴 스크롤 → **홈 화면에 추가**
4. 이름 (기본: 교육심리) 확인 → **추가**
5. 홈 화면에 "교육심리" 아이콘 생성
6. 아이콘 누르면 풀스크린으로 앱처럼 열림
7. 같은 기능들 확인 (이전/다음/즐겨찾기/검색/번역)

> 처음 한 번은 인터넷 필요. 한 번 캐시되면 비행기 모드에서도 열립니다 (Service Worker).

## 8. 매년 데이터 갱신 (이후 반복 작업)

매년 1편을 새로 다시 채점하기 위해:

```powershell
cd D:\AI_Root\cherin
python scripts\refresh_data.py
```

이게 자동으로 처리:
- OpenAlex 에서 top-500 다시 받음
- age-normalized 점수 재계산 (현재 연도 기준)
- top-365 추출
- `app/src/main/assets/papers.json` (Android 용) + `docs/papers.json` (PWA 용) 동시 갱신

push:
```powershell
git add app/src/main/assets/papers.json docs/papers.json
git commit -m "data refresh $(date +%Y-%m)"
git push
```

push 후 1~2분이면 PWA URL 에 새 데이터가 자동 반영됩니다 (서비스 워커가 새 papers.json 을 받아옴).

Android 앱은 별도 APK 빌드 + 재설치 필요 (`./gradlew assembleDebug`).

## 9. 문제가 생기면

가장 흔한 케이스:

| 증상 | 원인 | 해결 |
|---|---|---|
| Pages URL 이 404 | `/docs` 경로 미선택 | Settings → Pages → Folder 다시 `/docs` |
| 페이지가 빈 화면 | papers.json 누락 | `git ls-files docs/papers.json` 확인 |
| 별 토글이 새로고침 후 사라짐 | localStorage 차단 (시크릿 모드 등) | 일반 창에서 사용, "홈 화면에 추가" 권장 |
| iPhone Safari 가 캐시된 옛 버전 표시 | 서비스 워커 stale | 설정 → 사파리 → 방문 데이터 지우기 또는 PWA 의 "데이터 새로고침" 버튼 |
| 번역 새 창이 안 뜸 | 팝업 차단 | iOS 설정 → Safari → 팝업 차단 끄기 |
